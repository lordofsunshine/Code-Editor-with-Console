Inactive Accounts and Projects

Chat Messages

Project chat messages are subject to automatic cleanup:

  • Chat conversations that remain inactive for 24 hours (with no new messages) will be automatically deleted.
  • If a project's chat receives new messages within the 24-hour period, the cleanup timer resets.
  • This policy helps maintain database performance and ensures that only active conversations are retained.

We recommend regularly backing up important code and data. The platform is designed for active collaboration, not long-term archival storage.

3. Collaboration and Permissions

Our platform supports real-time collaboration with role-based access control:

  • Viewer: Read-only access to project files. Cannot create, edit, or delete files.
  • All collaborators can participate in project chat conversations.
  • Project owners can remove collaborators at any time.
